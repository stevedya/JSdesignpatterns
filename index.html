Design Patterns

A easy guide to design patterns in JavaScript

What are patterns

Patterns are tried and tested reusable techniques that assist in solving common programming problems. The patterns are rigorously tested and analyzed by developers to ensure they are a trusted solution. They must, at a minimum, meet these three requirements: fitness of purpose, usefulness and applicability to be considered valid (Addy Osmani, 2017). Although patterns are not one-size-fits-all, they help create efficient and reliable solutions based on proven techniques.

Design patterns in JavaScript

Design patterns are utilized everyday in JavaScript and JavaScript libraries. You may not have realized it but many programming languages, including JavaScript and jQuery, have design patterns already built-in. We use patterns in DOM selectors, even though document.getElementByClassname(“.item”) is more efficient in the browser, we still lean heavier towards the jQuery library where we can select the same item by writing $(“.item”). Design patterns in JavaScript are typically used in large-scale applications but can assist programs of any size in solving common issues and creating readable, reliable and maintainable code.

The history and importance of design patterns

Design patterns in programming have been used since the creation of the programming industry. The techniques were based on the work of architect Christopher Alexander and became widely known and accepted after a now iconic publication about design patterns for software engineering in 1995. Design patterns are important because they can be used on recurring problems in application development making the program more efficient. They are also important because they bring “method to the madness”. Design Patterns are trusted, reliable solutions that also assist in creating meaningful conversations between developers.

Constructor Pattern

Category: Creational + Builder

Creator: Richard Cornford and others

What it is used for

The constructor pattern is used to create unique type of objects with certain traits. It prepares the object for use allowing to pass arguments where the constructor can set the values of the properties and methods to make a new object. “When we call a JavaScript constructor to create an object, all the properties of the constructor's prototype are then made available to the new object. In this fashion, multiple Car objects can be created which access the same prototype. “ (Addy Osmani, 2017)

Problems it solves

The Constructor pattern allows us to construct a class-like object in JavaScript which is a classless language.

Pros & Cons

Pros
  • “Classes” in JavaScript
  • Highly efficient when all properties are known
  • Any changes affects only that instance
  • Makes applications more efficient and structured
  • Widely known pattern makes it easy for other developers to understand
  • Constructor can make multiple objects rather than one with an object literal
  • The prototype can be shared in a public method, but the constructed object will have closure (Sam Selikoff, 2013)
Cons
  • Any changes affects only that instance
  • Performance issues compared to other patterns
  • If the new keyword is not used or forgotten variables will be created in the global scope and won’t behave as expected

Other Info

They have frameworks that replace the native javascript framework for constructors ex. Backbone.js, ember.js, etc.
